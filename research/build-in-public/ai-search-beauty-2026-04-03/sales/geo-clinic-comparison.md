# GEO 감사 비교 — 뮤즈 명동 vs 샤인빔 강남

Date: 2026-04-03

---

## 점수 비교

| 항목 | 뮤즈 명동 (~28점) | 샤인빔 강남 (~35점) |
|------|-------------------|---------------------|
| robots.txt | ✅ 전체 허용 | ✅ 합리적 차단 |
| sitemap.xml | ⚠️ 5개 URL, 2021년 | ✅ 175개 URL, 최신 |
| Schema.org | ❌ 없음 | ❌ 없음 |
| llms.txt | ❌ 없음 | ❌ 없음 |
| 메타 태그 | ❌ 없음 | ⚠️ title+og만 (desc 없음) |
| JS 렌더링 의존 | ❌ 심함 | ❌ 심함 (Next.js) |
| AI-citability 블록 | ❌ 없음 | ❌ 없음 |
| 가격 텍스트 | ✅ 60+ 시술 명시 | ⚠️ 카테고리만 확인 |
| 연락처/주소 | ✅ 명확 | ✅ 명확 |
| 예약 CTA | ✅ 있음 | ✅ 있음 |
| 다국어 | ✅ 4개 | ✅ 5개 |

---

## 공통 문제점 (= 우리가 해결할 것)

1. **Schema.org 구조화 데이터 0** — 두 곳 모두
2. **llms.txt 없음** — 두 곳 모두
3. **JS 렌더링 의존** — AI 크롤러가 콘텐츠 못 읽을 가능성
4. **AI-citability 콘텐츠 없음** — 비교 쿼리에 인용될 팩트 블록 없음

→ 이 4가지가 GEO 서비스의 **핵심 작업 범위**

---

## 클리닉별 영업 포인트

### 뮤즈 명동 — "정보는 풍부한데 포장이 없다"
- 강점: 60+ 시술/가격이 텍스트로 있음 → 구조화만 하면 즉시 효과
- 약점: sitemap 5년 방치, 메타 태그 0
- 영업 메시지: "사장님, 가격 정보를 이렇게 잘 올려놓으셨는데 AI가 못 읽고 있습니다"

### 샤인빔 강남 — "기술은 있는데 AI한테 안 보인다"
- 강점: sitemap 175개, 5개 언어, 투명성 철학
- 약점: Next.js 렌더링으로 AI 크롤러 접근 제한
- 영업 메시지: "사장님, 투명하게 공개하신 정보를 AI가 실제로 못 읽고 있습니다"

---

## 작업 우선순위 (두 곳 공통)

```
[즉시] Schema.org JSON-LD 삽입 → AI가 "이건 클리닉, 이건 시술, 이건 가격" 인식
[즉시] llms.txt 생성 → AI에게 사이트 소개 문서 제공
[1주] AI-citability 팩트 블록 → 비교 쿼리에 인용될 수 있는 콘텐츠
[협의] SSR/정적 HTML → JS 의존 해소 (기술적 난이도 높음)
```

## 세일즈 시 사용할 비교 데이터

아웃리치 메시지에 첨부:
1. 두 클리닉의 GEO 점수 (28 vs 35 vs 업계 평균 대비)
2. ChatGPT/Gemini 실제 검색 스크린샷
3. "구조화 데이터 없음" 한 줄 팩트
4. 경쟁사 대비 위치 (아직 아무도 안 하고 있다는 점이 기회)
