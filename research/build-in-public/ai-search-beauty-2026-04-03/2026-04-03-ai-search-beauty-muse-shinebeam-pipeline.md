## Step-by-step pipeline

## Phase 0 — Outreach-ready asset pack (same day)
### Done / prepared
- interview script created
- pilot offer created
- target scorecard created
- two target briefs prepared

### Must be ready before sending first message
- 2 target-specific first-contact messages
- 1 short call opener
- 1 one-paragraph follow-up pitch
- 1 intake checklist for URLs
- 1 fact approval format

Exit criterion:
- you can message both targets today without improvising the core offer

---

## Phase 1 — First contact (day 1)
Channel priority:
1. official Kakao / 상담 channel if visible
2. phone call to front desk
3. website inquiry form / reservation note
4. Instagram DM only if official business channel is active

### Contact cadence
#### Touch 1
- short text or call opener
- ask for permission to send one concise explanation

#### Touch 2
- send 3–5 sentence explanation
- ask for one small next step only: “official URLs 2–3개만 보내주실 수 있을까요?”

#### Touch 3
- if no reply in 2–3 business days, send one follow-up with one concrete proof example and low-pressure CTA

Exit criterion:
- at least one target responds, or both go silent after 3 touches

---

## Phase 2 — Intake (day 1–3)
If a target replies positively, ask for:
1. official homepage URL
2. official pricing or event URL
3. booking/contact URL
4. branch page URL if separate
5. preferred treatment to test first

### Keep intake friction low
Do not ask for a form first.
Use a plain message:
"URL 2~4개만 주시면 저희가 초안을 만든 뒤, 공개 전 검수만 부탁드리겠습니다."

Exit criterion:
- at least 2 useful URLs received

---

## Phase 3 — Source-grounded fact extraction (same day after intake)
Assistant work:
1. capture source URLs
2. extract exact facts only
3. separate:
   - explicitly stated
   - not clearly stated
4. avoid claims expansion
5. draft structured profile fields

### Output format
- Provider name
- Branch
- Treatment category
- Publicly visible price / package details
- Publicly visible notes / restrictions
- Booking path
- Contact details
- Last verified date
- Source URL list

Exit criterion:
- one clean fact sheet ready for provider review

---

## Phase 4 — Draft profile page + approval packet (day 1–4)
Assistant prepares:
1. public profile draft
2. side-by-side fact sheet
3. approval message

### Approval message goal
Make approval feel easy:
- “틀린 부분만 말씀 주세요” instead of “전체 다시 써주세요”
- approval should be one short pass

Exit criterion:
- provider says one of:
  - approved
  - approved with edits
  - hold / not now

---

## Phase 5 — Publish (day 2–5)
Publish profile page with:
- plain HTML text
- structured data
- source citations
- CTA to official booking/contact page
- real robots/sitemap inclusion
- analytics tags for CTA clicks

Exit criterion:
- public URL live and crawlable

---

## Phase 6 — Before/after query logging (week 1 onward)
For each target, run fixed query set:
- [brand name] + 제모 가격
- [district] + 겨드랑이 제모 추천
- [district] + 제모 가격
- [clinic name] + 겨드랑이 / 인중 / 레이저 제모

Capture on each platform:
- ChatGPT
- Gemini
- Perplexity
- Google AI/search where available

Track:
- appearance Y/N
- source cited
- factual completeness
- distortion/errors

Exit criterion:
- dated before/after evidence log exists even if results are mixed

---

## Phase 7 — Pilot decision
Success for this two-target wave means any of:
- 1 clinic agrees to pilot
- 1 clinic sends URLs and approves a draft
- 1 clinic asks pricing / next steps seriously
- 1 live page shows directional improvement in query tests

If both fail for the same reason, classify the failure:
- wrong target
- wrong offer wording
- wrong outreach channel
- weak proof / too abstract
