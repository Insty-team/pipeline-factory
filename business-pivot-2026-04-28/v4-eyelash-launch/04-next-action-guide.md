## 3. 사용자가 미리 준비할 것 (3개)

### 준비물 1 (필수, Day 3 전): **시술 사진 1장**

- 용도: M1+M2 통합 워크플로우 데모 (사장에게 "사진 1장 카톡 → 인스타+블로그 자동" 보여주기)
- 어디서: 유어라인 인스타 ure.line 게시물 캡쳐 1장 또는 사용자 본인이 시술 받은 사진 (있으면)
- 워터마크 없는 깨끗한 원본이면 더 좋음 (없으면 워터마크 있어도 됨)
- 저장 경로: `Day1_data_collection/sample_photo.jpg`

### 준비물 2 (선택, Day 3 카드뉴스): **디자인 도구 결정**

3가지 중 1개 선택:
- (a) **Canva 무료** (추천 — 빠름, 템플릿 풍부, 카드뉴스 1세트 1~2h)
- (b) **Figma 무료** (디자인 익숙하면)
- (c) **외주** (크몽 1세트 5~10만)
- (d) **나중에** (텍스트 카피만 먼저 시제품에 넣고, 디자인은 사장과 합의 후)

→ 결정 안 해도 Claude가 텍스트 카피는 작성 가능. Day 5 DM 시점에 텍스트만 보여줄지 디자인까지 보여줄지 차이.

### 준비물 3 (Day 5 DM 직전): **인스타 로그인**

- ure.line에 DM 보낼 사용자 계정 — 평소 운영 중인 인스타 계정으로 로그인 상태 확인
- 새 계정이면 너무 신규 (스팸 의심 risk) — 7일+ 운영된 계정이 좋음
- 프로필 사진·바이오 1줄이라도 있으면 답장률 ↑

## 7. 4주 베타 시작 후 (참고만 — 미래)

DM 답장 → 미팅 → 베타 시작되면 매주 월요일 아침 사용자가 사장에게 카톡:

```
[유어라인 X주차 리포트]
지난주:
- 인스타 게시 4건, 도달 +XX%
- 후기 답글 X건 자동
- 알림톡 발송 XX건
- 노쇼 X건 (전월 동기 대비 -X건)
- 신규 DM 문의 X건

이번주 계획:
- 블로그 1편
- 디자인 카드뉴스 인스타 게시
- ...

원장님 피드백 한 줄만 부탁드려요.
```

→ Claude가 매주 위 텍스트 자동 생성. 사용자는 복붙만.

## 8. 막힐 때 / 자주 물어볼 것 (FAQ)

**Q1. 사장이 인스타 DM 안 보면?**
→ 02 §7 위험 시나리오 표 참조. 카카오 → 블로그 비밀댓글 → 전화 (3단계).

**Q2. 시제품 보고 "이미 비슷한 거 한다"고 하면?**
→ "어떤 부분이 비슷한지 보여주실 수 있나요?" — 빈자리 다시 발견 후 재제안.

**Q3. Claude한테 다음에 뭐 시켜야 할지 모르겠다?**
→ 이 문서 §6 "오늘 추천 순서" 다시 보기. 또는 "다음에 뭐 해야해?" 1줄 보내면 답해줌.

**Q4. v3 (네일·에스테틱) 진행 안 하나?**
→ 별도 트랙. 유어라인 베타 결과 (4주) 본 뒤 결정. 현재는 v4 집중.

**Q5. 5곳 무료 시제품은 언제?**
→ 유어라인 1순위 끝나고 결과 검증 후. 꼬모래쉬 → 다른 후보 순서. (02 문서 §10 = 01 문서 §5 시나리오 참조)
