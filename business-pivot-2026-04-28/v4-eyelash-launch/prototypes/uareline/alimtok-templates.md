## 1. 자동화 전체 워크플로우 (1장 다이어그램 텍스트)

```
┌─────────────────────────────────────────────────────────────────────────┐
│   사장 일상 작업 (현재)                  →                자동화 (베타)            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  예약 들어옴                                예약 들어옴                       │
│   ↓ (사장 카톡 수동 답변)                    ↓ (예약 시스템 → 자동 트리거)        │
│  변경 요청 응대 (X분)                        A1 예약 확정 알림톡 (자동)            │
│   ↓                                         ↓                            │
│  방문 전날 D-1 ··· (사장 잊거나 못 보냄)        A2 1일 전 알림톡 (자동, 18시)       │
│   ↓                                         ↓                            │
│  방문 당일 D-Day                             A3 당일 알림톡 (자동, 9시)           │
│   ↓                                         ↓                            │
│  시술 → 끝                                   시술                           │
│   ↓                                         ↓                            │
│  D+1 후기 요청 (X — 한 번도 못 보냄)           A4 D+1 후기 요청 (자동, 다음 날)    │
│   ↓                                         ↓                            │
│  D+28 재방문 알림 (X — 단골 cycle 깨짐)       A5 D+28 재방문 알림 (자동)          │
│                                                                         │
│  -------------------- 예외 시나리오 --------------------                    │
│                                                                         │
│  노쇼 발생 → 사장 직접 연락                  C1 노쇼 follow-up (자동, 저녁)       │
│  변경 요청 → 카톡 일일이                     C2 변경 응답 템플릿 (자동, 즉시)      │
│  6주+ 미방문 단골 모니터링 X                  D3 이탈 단골 안부 (자동 주간 1회)     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘

   📊 사장 노동: 약 30분/일 (수동 답변·후기 요청·재방문 알림) → 0분/일 (자동)
   📈 노쇼 -30~50% / 4주 재방문률 +15~30% / 후기 작성률 +20~40% (추정)
```

## 2. 카카오 알림톡 사전 승인 절차 (KakaoTalk for Business)

### 2-0. 알림톡 vs 친구톡 vs 일반 메시지 — **꼭 알림톡으로 등록 필요**

| 종류 | 가격 | 사전 승인 | 친구추가 필요 | 발송 가능 시간 |
|---|---|---|---|---|
| **알림톡** ⭐ | **8.5원/건** | **필요 (3영업일)** | ❌ X — 전화번호만 있으면 발송 | 24h 가능 |
| 친구톡 | 12~18원/건 | X | ✅ 필요 | 24h 가능 |
| SMS | 10~20원/건 | X | X | 야간 제약 |

→ 유어라인은 **모든 단골 전화번호 보유** (회원권 DB) → **알림톡이 최적**.

### 2-1. 사전 절차 (사장 입장에서 1회 셋업)

```
[Step 1] 카카오톡 채널 (구 플러스친구) 개설 ─ 사장 본인 인증
   ↓ (사장 1회, 5분)
   - 카카오톡 → 비즈니스 채널 → 새 채널 만들기
   - 사업자등록증 + 신분증 업로드
   - 채널명 "유어라인" + 카테고리 "뷰티/속눈썹"

[Step 2] 알림톡 발송 사업자 (이하 "발신 프로파일") 등록
   ↓ (사장 1회, 영업일 1~2일 검수)
   - https://business.kakao.com → 알림톡 → 발신 프로파일 신청
   - 사업자등록증 (필수)
   - 채널 연동 (Step 1)
   - 검수 통과 → 발신 프로파일 ID 발급

[Step 3] 알림톡 템플릿 12종 등록
   ↓ (사장 + 우리 1회, 영업일 ~3일 검수)
   - 발송 솔루션 선택 (3가지 중 1):
       (a) ⭐ 알리고 (Aligo) — 가장 보편, API 가장 쉬움, 월 무료 1000건
       (b) 비즈M (BizM) — 카카오 공식 솔루션 파트너
       (c) 솔라피 (SOLAPI) — 한국 SaaS, 다양한 채널 통합
   - 솔루션 가입 → 발신 프로파일 연동
   - 12종 템플릿 입력 (변수 표시: #{고객명}, #{날짜시간}, #{메뉴} 등)
   - 카카오 검수 신청 (각 템플릿마다 사유·발송 시점 명시)
   - 검수 통과 → 템플릿 ID 발급

[Step 4] 발송 자동화 셋업 (우리 1회, 1일)
   - 트리거 시스템 (예약 DB + cron) → 솔루션 API 호출
   - 변수 매핑 (예약 데이터 → 템플릿 변수)
   - 테스트 발송 5건 (사장 본인 폰 → 톤 확인)

[Step 5] 운영 시작
   - 베타 4주 동안 매일 자동 발송 + 사장 확인 (1일 1회 카톡 1줄)
```

### 2-2. 카카오 검수 통과 핵심 (실패 사례 회피)

| 검수 기준 | OK 예시 | NG 예시 |
|---|---|---|
| **정보성**: 광고 X | "예약 확정 안내드려요" | "오늘 특별 할인!" ❌ |
| **변수 사용**: #{변수}만 정확 | `#{고객명}님, #{날짜시간} 예약` | "고객님 예약" (변수 X) → 검수 실패 |
| **변경/취소 안내 1줄 필수** | "변경: 010-4908-6511" ✅ | (없음) → 실패 |
| **이모지·특수문자**: 1~2개만 OK | ♡, 📅, 💄 | ✨🌟❗❗❗ 과도하면 ❌ |
| **광고 표시**: D-3 메디핑크는 "광고/친구톡" 분리 | (D1·D2은 친구톡으로 분리) | 알림톡에 "할인 50%" → 즉시 반려 |
| **D+28·D+42 알림**: 친구톡 (재방문 권유는 마케팅성) | 알림톡 X / 친구톡 발송 | 알림톡으로 보내려 하면 검수 ❌ |

### 2-3. 12종 템플릿 분류 (정보성 vs 광고성)

| 템플릿 | 분류 | 발송 채널 | 검수 난이도 |

| A2 1일 전 | **정보성** | 알림톡 | 쉬움 |
| A3 당일 | **정보성** | 알림톡 | 쉬움 |

| A5 D+28 재방문 알림 | **광고성** (재방문 권유) | **친구톡 권장** | 알림톡 시도하면 반려 가능 |
| B1·B2·B3 단골 환영 | **정보성** (예약 확정의 변형) | 알림톡 OK | 쉬움 |

| D3 이탈 단골 안부 | ⚠️ **경계** (안부는 정보성, 광고적 표현은 NG) | 친구톡 권장 | 중간 |

## 5. 자동화 코드 스켈레톤 (참고)

### 5-1. 알리고 API 호출 (Python)

```python
import requests
from datetime import datetime

ALIGO_API_KEY = "<발급받은 키>"
ALIGO_USER_ID = "uareline_owner"
SENDER = "01049086511"  # 발신 프로파일

def send_alimtok(template_id: str, receiver: str, variables: dict):
    """카카오 알림톡 발송. 사전 승인 받은 템플릿만 가능."""
    msg_filled = render_template(template_id, variables)
    res = requests.post(
        "https://kakaoapi.aligo.in/akv10/alimtalk/send/",
        data={
            "apikey": ALIGO_API_KEY,
            "userid": ALIGO_USER_ID,
            "senderkey": "<채널 발신 프로파일 키>",
            "tpl_code": template_id,  # 카카오 검수 통과 후 발급된 ID (예: TT_0001)
            "sender": SENDER,
            "receiver_1": receiver,
            "subject_1": "유어라인 알림",
            "message_1": msg_filled,
            "testMode": "Y" if BETA_MODE else "N",
        }
    )
    return res.json()


# 트리거 예시 (cron 18:00 매일 — A2 1일 전)
def cron_alimtok_d_minus_1():
    tomorrow = today() + timedelta(days=1)
    bookings = db.query("SELECT * FROM bookings WHERE date = ?", tomorrow)
    for b in bookings:
        send_alimtok(
            template_id="TT_A2_DM1",
            receiver=b.phone,
            variables={
                "고객명": b.name,
                "시간": b.time,
                "메뉴": b.menu,
                "전화": "010-4908-6511",
            },
        )
        log_to_dashboard("alimtok_a2_sent", b.id)
```

### 5-2. cron 셋업 (서버 또는 GitHub Actions)

```yaml
# .github/workflows/uareline-alimtok-cron.yml
name: 유어라인 알림톡 cron

on:
  schedule:
    - cron: "0 9 * * *"   # 매일 9시 (KST 18시) → A2 1일 전
    - cron: "0 0 * * *"   # 매일 0시 (KST 9시) → A3 당일
    - cron: "0 2 * * *"   # 매일 2시 (KST 11시) → A4 D+1 / A5 D+28 / D2 D+42 / D3 6주+

jobs:
  send_alimtok:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pip install requests
      - run: python scripts/cron_alimtok.py --slot ${{ github.event.schedule }}
        env:
          ALIGO_API_KEY: ${{ secrets.ALIGO_API_KEY }}
          NAVER_BOOKING_API: ${{ secrets.NAVER_BOOKING_API }}
```

## 6. 자주 묻는 질문 (FAQ)

### Q1. 네이버나 카카오 비즈니스 등에 등록하면 이런 자동화 기능 빌트인으로 제공되는 거 없나? 직접 구현이 최선인가?

**짧은 답**: 빌트인만으로는 70% 부족. 외부 솔루션 (알리고 등) 필수.

**상세 표**:

| 플랫폼 | 빌트인으로 OK | 안 되는 것 (외부 솔루션 필요) |
|---|---|---|
| **네이버 예약** | 예약 확정 SMS/카톡 / 1일 전 알림 (네이버 표준 톤) | 사장 톤 커스텀 X / D+1 후기·D+28 재방문 자동 X / 메디핑크 캠페인 자동 X |
| **카카오 채널** | 사장 직접 메시지 작성·발송 (수동) / 친구 등록 단골 일괄 발송 | **트리거 자동화 X** ("예약 → 자동 발송" X / "D+28 → 자동 알림" X) |
| **카카오 비즈니스** | 알림톡·친구톡 인프라 (발신 프로파일) | **자동화 발송은 외부 솔루션 통해서만** |
| **카카오 i 오픈빌더** | 챗봇 (대화형) — DM 자동 응답 | 시간/이벤트 트리거 발송 X |

**대안 — 뷰티 1인샵 통합 시스템**:

| 솔루션 | 빌트인 자동화 | 월 가격 |
|---|---|---|
| **메리트** (https://merit.salon) | 예약 + 알림톡 + 단골 DB + LTV + 회원권 | ~5~10만/월 |
| **뷰티시스템** | 위 동일 | ~5~12만/월 |
| **솔트시스템** | 위 동일 | ~6만/월 |
| **알리고 + 자체 코드** ⭐ | API 자동화 (직접 트리거) | **무료 (1000건/월)** ← 우리 채택 |

**우리가 알리고 채택한 이유**:
- 사장 이미 네이버 예약 + 카카오 채널 사용 중 → 시스템 변경 거부감 ↓
- 메리트는 예약 시스템 도입 강요 = 학습 비용 ↑
- 알리고는 **알림톡 자동화 레이어만 추가** (사장 운영 그대로)
- 베타 4주 = 알리고 무료 안에 완료
- 정기 운영 시점에 메리트로 이전 가능 (commission 협상 후 사장 의향)

→ 알리고 + 알림톡 = 0원 / 메리트 = 월 5~10만 (사장 부담)

### Q2. 알림톡, 친구톡, 알리고 솔루션의 차이가 뭔지? 알림톡·친구톡은 카톡 솔루션이고, 알리고는 또 다른 솔루션인가?

**짧은 답**: **알림톡·친구톡 = 카카오의 "메시지 형식"** / **알리고 = 외부 발송 대행 SaaS**.

```
🏢 카카오 (KakaoTalk for Business)
   └─ 카카오톡 메시지 형식 2종
       ├─ 알림톡 (정보성, 8.5원/건, 사전 승인 필요)
       └─ 친구톡 (광고성, 12~18원/건, 친구 추가 필요)

📡 발송 솔루션 (알림톡·친구톡을 자동으로 보내려면 사용)
   ├─ 알리고 (Aligo) ⭐ 우리 채택 — 무료 1000건
   ├─ 비즈M (BizM) — 정액제 5만/월
   ├─ 솔라피 (SOLAPI) — 1만/월부터
   └─ 기타 카카오 공식 파트너
```

**비유**:

```
[카카오톡 = 우체국]
   ├─ 알림톡 = 등기우편 (정보성, 신원 확인·검수 필요)
   └─ 친구톡 = 광고 전단지 (마케팅, 친구 등록한 사람만)

[알리고 = 우체국 대행 서비스]
   - 사장이 일일이 우체국 가기 귀찮음 → 알리고에 위탁
   - 알리고가 "우체국에 대신 가서" 등기우편/전단지 보내줌
   - 카카오 + SMS + 이메일 등 여러 채널 동시 처리 가능
   → 카카오 대체가 아니라 "위에 얹는 자동화 레이어"
```

**핵심 차이 표**:

| 항목 | 알림톡 | 친구톡 | 알리고 |
|---|---|---|---|
| 분류 | 카카오 제품 | 카카오 제품 | 외부 솔루션 |
| 무엇 | 메시지 형식 | 메시지 형식 | 발송 대행 SaaS |
| 친구 등록 | ❌ 불필요 (전화번호만) | ✅ 필요 | (메시지 형식 따라감) |
| 검수 | ✅ 카카오 사전 승인 (3영업일) | ❌ 불필요 | (메시지 형식 따라감) |
| 광고 가능? | ❌ 정보성만 | ✅ 광고 OK | (메시지 형식 따라감) |
| 가격 (건당) | 8.5원 | 12~18원 | 카카오 가격 + 알리고 마진 (베타 무료) |
| API 직접 연결 | 가능하지만 복잡 | 가능하지만 복잡 | **알리고 1개 API로 둘 다 발송** |

**알리고가 필요한 이유 (왜 카카오 직접 API X)**:

| 직접 카카오 API | 알리고 |
|---|---|
| 발신 프로파일 관리 본인이 (복잡) | 알리고가 일괄 관리 |
| 템플릿 검수도 본인이 신청 | 알리고 대시보드에서 등록 → 자동 검수 |
| API 인증·서명 구현 직접 | 알리고 API key 1개면 끝 |
| SMS·이메일은 별도 통합 | 알리고 1개로 다 처리 |
| 대시보드·로그 X | 알리고 대시보드 빌트인 |

→ 1인 사장이 카카오 API 직접 = 1주일 학습 / 알리고 = 30분 셋업.
