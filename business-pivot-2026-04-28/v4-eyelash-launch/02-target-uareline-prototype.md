## 2. 강점·약점 매핑

### 강점 (이미 잘 함 — 효율화 필요)

| 강점 | 현재 사장 노동 (추정) | 풀패키지 적용 후 | 가치 |
|---|---|---|---|
| 인스타 매주 게시 | 3~4h/주 | **30분/주** (사장은 시술 사진만 카톡, 캡션·해시태그·게시 자동) | 시간 ↓ + 사장 톤 보존 |
| 블로그 4~5일 간격 | 2h/포스트 | **30분/포스트** (AI 초안 → 사장 편집) | 시간 ↓ |
| 시술 실력 (별점 4.93) | — | 콘텐츠 quality에 시술 강점 반영 | 마케팅 효과 ↑ |
| 메뉴 11종 다양성 | 신규 매번 30분 컨설팅 | **0분** (디자인 카드뉴스 1세트로 흡수) | 30분/신규 ↓ |

### 약점 (안 함 — 채움)

| 약점 | 매출 손실 추정 (월) | 풀패키지 적용 |
|---|---|---|
| 후기 답글 X | 단골 1명 LTV 200만 × 이탈률 5%/월 = ~50만 (후기 응대 부재 영향) | M6: 자동 답글 (사장 톤 보존) |
| 4주 재방문 알림 X | 단골 이탈 10~20% × 단골수 = 월 매출 -10~30% | M5: 자동 알림톡 |
| 카카오 자동응답 X | 시술 중 DM 놓침 → 신규 예약 손실 (월 5~10건) | M4: GPT 챗봇 |
| 통합 리포트 X | 데이터 의사결정 부재 | M10: 월간 리포트 |
| 단골 LTV 분석 X | 이탈 위험 단골 발견 못 함 | M9: CRM |

### 통합 자체의 가치

5개 채널 (인스타·블로그·플레이스·카카오·후기) → **1개 운영 보드**.
사장은 **시술 사진 1번 카톡 = 끝**.

## 3. 모듈별 산출물 + 효용성 측정 시스템

### 핵심 원칙: 측정 가능한 것만 commission

**측정 시스템 = 시제품에 내장**. 베타 4주 동안 **사장이 추가 작업 0** + 자동 데이터 누적.

### 모듈별 (시제품 단계 → 베타 운영 단계 → 측정)

| # | 모듈 | 시제품 (DM 발송 전) | 베타 운영 중 측정 | Commission 계산 |
|---|---|---|---|---|
| **M1** | 인스타 운영 + AI 캡션 | 캡션 5종 데모 (사장 톤) + 해시태그 30개 | 인사이트 도달·신규 팔로워·DM 문의 / 신규 손님 매핑 | 신규 손님 1명당 2~5만 |
| **M2** | 블로그 AI 초안 | 초안 1편 (사장 30분 편집 가능) | 검색 유입 (네이버 SEO) + 신규 손님 ("어디서 보고?") | 검색 유입 신규 1명당 3~5만 |
| **M3** | 네이버 플레이스 + 후기 답글 | 최근 후기 5~10건 답글 작성 (사장 톤) | 신규 후기 +건수, 별점 유지 | (M1·M6과 합쳐서) |

| **M10** | 월간 통합 리포트 | 1장 (현재 → 적용 시 추정) | 월말 자동 생성 | 0원 (의사결정 가속 가치) |

## 6. 4주 베타 운영 매뉴얼

### 시작 시 사장과 합의할 것 (반드시 카톡 캡처)

1. **KPI 4개 측정 동의**
   - 노쇼 감소 (-30~50%)
   - 4주 재방문률 (+15~30%)
   - 신규 후기 +건수
   - 신규 DM 문의 +건수

2. **후기 1줄 약속** (베타 종료 시)

3. **베타 종료 시 옵션 4개 중 선택 약속**

4. **데이터 수집 SOP 동의**
   - 신규 손님께 "어디서 보고 오셨어요?" 1줄 질문
   - 단골 정보 입력 (사장 자율 — 없어도 측정 가능한 부분만)

### 4주 종료 시 결과 1장 + Commission 협상

> 3. **신규 후기**: X건 (전월 +Y건) → 신규 손님 유입 추정 ≈ M명

> 5. **인스타·블로그 신규 손님**: N명 → 매출 기여 ≈ XX만원
> 6. **단골 회복**: X명 → LTV ≈ XX만원
>
> **추정 4주 매출 기여 합계: XX만원**

## 7. 위험 + 백업 시나리오

| 상황 | 대응 |
|---|---|
| 사장이 인스타 DM 안 봄 (유어라인이 응답 빠르지 않음) | 카카오 채널 → 블로그 비밀댓글 → 전화 (3단계) |

| 4주 베타 중 사장이 협조 안 함 (사진 안 보냄) | 자동화 가능한 부분만 (M5·M6·M9)로 진행 + 데이터 부족 인정 |
| 4주 종료 시 commission 거절 | 옵션 4개 중 1개만 받고 정중히 종료 → 다음 후보 진행 (학습 자산 확보) |
| 측정 데이터 모호 (효용성 입증 약함) | 정성 데이터 (사장 인터뷰) + 다음 사이클 측정 인프라 강화 |
