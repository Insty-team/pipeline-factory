# 가설 신뢰도 점수 체계

> 출처: Insty-Marketing final-plan (2026-03-20)
> Pipeline Factory의 가설 평가에 적용

---

## 신뢰도 공식

```
신뢰도 = (수요 증거 × 0.30) + (경쟁 부재 × 0.25) + (해결 가능성 × 0.25) + (수익화 잠재력 × 0.20)

0.7 이상: 바로 실행
0.4~0.7: 추가 조사 후 판단
0.4 미만: 스킵
```

---

## 수요 증거 데이터 소스

| 데이터 소스 | 수집 방법 | 가설에 주는 것 |
|-----------|----------|-------------|
| Reddit/커뮤니티 글 | AI 톤 분석 | **소재**: 어떤 문제가 진짜 아픈지 |
| upvote + 댓글 수 | API 수집 | **수요 크기**: 공감하는 사람 수 |
| Google Trends 검색량 | API 수집 | **시장 크기**: 검색하는 사람 수 |
| 기존 도구 리뷰/별점 | 크롤링 | **경쟁 약점**: 어디를 찌를지 |
| 다채널 등장 빈도 | 교차 분석 | **보편성**: 여러 곳에서 나오면 진짜 |

## 검증 실험 매트릭스

| 영역 | 실험 방법 | 판단 데이터 |
|------|----------|-----------|
| 타겟 | 3가지 타겟 동시 테스트 | 응답률 비교 |
| 가격 | 3가지 가격 동시 제안 | 전환율 비교 |
| UX | 같은 PP × 3-5가지 UX 배포 | 사용률/재방문율 |
| 메시지 | 3종 A/B/C | 응답률 |
| 채널 | 동시 다채널 집행 | CPL 비교 |
