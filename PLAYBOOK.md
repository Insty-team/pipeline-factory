# Pipeline Factory 실행 플레이북

> Polsia 벤치마크 기반, "검증된 수익 파이프라인 발굴 서비스"
> 기간: 2026-03-28 ~ 2026-04-27 (4주)

---

## 서비스 철학

```
"영속적 사업 하나 찾는 것보다,
 필요할 때마다 작은 사업을 만들 수 있는 능력이 더 안전하다."

이 서비스는 도구가 아니라 "비즈니스 근육"을 만드는 과정이다.
```

### Polsia vs 우리의 본질적 차이

```
Polsia:  "AI가 당신 대신 회사를 운영합니다"
         → 유저는 수동적. 플랫폼 없으면 무력.
         → 복권 사는 구조. 대부분 $50 MRR 이하.

우리:    "작은 니치를 찾고 검증하는 법을 AI와 함께 반복하세요"
         → 유저는 능동적. 쓸수록 본인이 더 잘하게 됨.
         → 근육 만드는 구조. 반복할수록 성공률 상승.
```

### 자연스러운 종속 (강제 아닌 선택)

```
유저가 떠나지 않는 이유는 "못 떠나서"가 아니라:

1. 내 실험 히스토리가 여기 있음
   → 100번의 성공/실패 데이터 = 나만의 비즈니스 직관 DB
   → 다른 곳에서 처음부터 쌓기 싫음

2. 플랫폼 전체의 데이터를 누림
   → "이 니치는 지금 포화" 같은 정보는 혼자 못 얻음
   → 유저가 많을수록 내가 얻는 인사이트도 커짐

3. 내 근육이 이 도구에 최적화됨
   → 헬스장 바꾸면 기구 위치부터 다시 익혀야 하듯
   → "발견→검증→실행" 워크플로우가 몸에 밴 상태

4. 성공 경험이 쌓여있음
   → "여기서 3번째 파이프라인 성공했는데 왜 떠나?"
   → 성공한 기억이 있는 곳을 떠나기 어려움
```

### 셀링 메시지

```
대기업은 다음 먹거리 하나 찾는 데 수년, 수백억 씀.
당신은 $20와 48시간이면 됨.

큰 그림을 그리는 건 어렵지만,
작은 니치부터 찾아 만족시키는 연습은
인생살면서 가장 가치 있는 투자.
```

## 서비스 컨셉

```
"AI가 실행만 하는 게 아니라, 뭘 해야 돈이 되는지 찾아준다"
"니치 타이밍 머신 — 언제 들어가고 언제 나올지를 데이터로"

Input:  관심 분야 / 보유 자원
Output: 검증된 수익 파이프라인 (데이터 포함)
```

### 니치는 고갈되지 않는다

```
니치 생성 속도 > 니치 포화 속도

이유:
1. 기술/문화/규제 변화 → 새 니치 폭발적 생성
2. 프랙탈 구조 → 상위 포화 시 하위 니치 개방
3. 같은 니치도 시장별(한/중/미) 다른 문제

핵심 가치: "포화된 니치를 빨리 버리고, 새 니치를 빨리 잡는 것"
= 속도 게임 = 데이터 많은 플랫폼이 이김
```

## 가설 1000개 파이프라인

### 구조: 3개씩 세우고, 테스트하고, 인사이트 얻고, 반복

```
┌─────────────────────────────────────────────────────────┐
│                    가설 파이프라인 엔진                     │
│                                                           │
│  ┌─────────┐   ┌──────────┐   ┌──────────┐   ┌────────┐ │
│  │ 3개 가설  │──→│ 동시 테스트 │──→│ 분석/인사이트│──→│ 3개 가설 │─→ ...
│  │ 생성     │   │ (48~72h) │   │ 추출      │   │ 생성   │ │
│  └─────────┘   └──────────┘   └──────────┘   └────────┘ │
│       ↑                                           │      │
│       └───────────────────────────────────────────┘      │
│                    인사이트가 다음 가설에 반영               │
└─────────────────────────────────────────────────────────┘

1 사이클 = 3가설 × 72시간 = ~1주
목표: 1000가설 도달 = ~333 사이클 = ~6.4년 (1인)
     유저 100명이면 = ~8개월
     유저 1000명이면 = ~24일 ← 이게 플랫폼의 힘
```

### 사이클 상세 프로세스

#### Step 1: 가설 3개 생성 (AI 자동, 사람 승인)

```
Input:
  - 이전 사이클 인사이트
  - 현재 트렌드 신호 (크롤링 데이터)
  - 유저 관심 분야
  - 플랫폼 전체 데이터 (어떤 니치가 포화/블루오션인지)

Output (예시):
  가설 #001: "미국 시니어 대상 AI 가족사진 복원 서비스"
    - 신호: Reddit r/OldPhotos 구독자 6개월간 40%↑
    - 경쟁: 있지만 $50+ 고가, 저가 자동화 없음
    - 예상: $9.99/장, 월 200건 = $2,000 MRR
    - 검증 방법: 랜딩페이지 + Facebook 55+ 타겟 광고

  가설 #002: "한국 소상공인 AI 메뉴판 번역 서비스"
    - 신호: 네이버 "외국인 관광객 메뉴" 검색 300%↑
    - 경쟁: 수동 번역 업체만 있음 ($200+)
    - 예상: $29/월 구독, 식당 500개 = $14,500 MRR
    - 검증 방법: 네이버 플레이스 광고 + 샘플 3개 무료 제공

  가설 #003: "크로스보더 셀러용 AI 리뷰 번역/분석"
    - 신호: 아마존→쿠팡 역직구 셀러 커뮤니티 활성화
    - 경쟁: 범용 번역기뿐, 리뷰 특화 없음
    - 예상: $19/월, 셀러 300명 = $5,700 MRR
    - 검증 방법: 셀러 카페 홍보 + 무료 체험

사람 역할: 3개 중 감이 안 오는 건 교체 요청 가능
```

#### Step 2: 동시 테스트 (48~72시간, AI 자동)

```
각 가설에 대해 병렬 실행:

가설 #001                 가설 #002                 가설 #003
  │                         │                         │
  ├─ 랜딩페이지 생성         ├─ 랜딩페이지 생성         ├─ 랜딩페이지 생성
  ├─ 광고 3개 변형          ├─ 광고 3개 변형          ├─ 광고 3개 변형
  ├─ $20 광고 집행          ├─ $20 광고 집행          ├─ $20 광고 집행
  ├─ 72h 데이터 수집        ├─ 72h 데이터 수집        ├─ 72h 데이터 수집
  └─ 자동 리포트            └─ 자동 리포트            └─ 자동 리포트

총 비용: ~$60/사이클 (3가설 × $20)
```

#### Step 3: 분석 & 인사이트 추출 (AI 자동)

```
각 가설 결과:

가설 #001 결과:
  CTR: 4.1% ✅ (기준 2%+)
  이메일 수집: 38건 ✅ (기준 20+)
  CPC: $0.52
  판정: GO → Phase 2 (실제 서비스 빌드)

가설 #002 결과:
  CTR: 0.9% ❌
  이메일 수집: 4건 ❌
  CPC: $4.80
  판정: NO-GO
  패인 분석: 소상공인은 Facebook 안 씀. 채널 미스매치.

가설 #003 결과:
  CTR: 2.3% ⚠️
  이메일 수집: 15건 ⚠️
  CPC: $1.30
  판정: PIVOT → 가격/타겟 변경 후 재테스트

인사이트 추출 (다음 사이클에 반영):
  [I-001] 55+ 타겟 Facebook 광고가 예상보다 효과적 → 시니어 니치 추가 탐색
  [I-002] 한국 소상공인은 네이버/카카오 채널이 필수 → Meta 광고 비효율
  [I-003] 크로스보더 셀러는 관심은 있으나 $19 가격 저항 → $9 테스트 필요
  [I-004] 3개 중 1개 GO = 33% 성공률 → 현재 벤치마크로 기록
```

#### Step 4: 다음 3개 가설 생성 (인사이트 반영)

```
이전 인사이트 [I-001]~[I-004] 반영:

가설 #004: 시니어 + 다른 문제 (I-001에서 파생)
  "AI 손자 사진 캘린더 자동 생성 서비스"

가설 #005: 소상공인 + 올바른 채널 (I-002 PIVOT)
  "카카오 채널 기반 AI 메뉴판 → 네이버 플레이스 광고로 검증"

가설 #006: 완전 새로운 신호 기반
  "이번 주 Reddit 급등 키워드에서 발굴"

→ Step 1로 돌아감
```

### 인사이트 축적 구조

```
가설 #001~#003  → 인사이트 [I-001]~[I-004]
가설 #004~#006  → 인사이트 [I-005]~[I-009]
가설 #007~#009  → 인사이트 [I-010]~[I-013]
...
가설 #100       → 인사이트 ~130개 축적
가설 #1000      → 인사이트 ~1300개 축적

인사이트 DB 구조:
{
  id: "I-001",
  cycle: 1,
  category: "channel",       // channel, pricing, audience, niche, timing
  insight: "55+ 타겟은 Facebook이 Instagram보다 3x 효과적",
  confidence: 0.7,           // 검증 횟수에 따라 상승
  validated_count: 1,        // 이 인사이트가 맞았던 횟수
  contradicted_count: 0,     // 틀렸던 횟수
  derived_from: ["H-001"],   // 어떤 가설에서 나왔는지
  applied_to: ["H-004"],     // 어떤 가설에 적용됐는지
  market: "US",
  tags: ["senior", "facebook", "advertising"]
}

→ 이 DB가 곧 플랫폼의 핵심 자산
→ 유저가 많을수록 인사이트 축적 가속
→ 새 유저는 Day 1부터 1000개 인사이트 혜택
```

## 비즈니스 가설 테스트 (서비스 자체에 대한 가설)

### Batch 1: "사람들이 진짜 돈을 내는가?"

```
가설 B-001: 콘텐츠 → 전환
  "AI 파이프라인 실험 과정을 Twitter에 공개하면
   팔로워 중 2%가 파운딩 멤버로 전환한다"
  테스트: Twitter 스레드 3개 발행, 링크 클릭률 + 전환 측정
  GO 기준: 팔로워 500명 중 10명 전환 (2%)
  비용: $0 (시간만)

가설 B-002: 가격 민감도
  "월 $29에 '주 1회 검증된 니치 리포트'를 구매한다"
  테스트: 랜딩페이지 A/B ($29 vs $49 vs $19)
  GO 기준: $29에서 전환율 3%+
  비용: $30 (광고)

가설 B-003: 핵심 가치 메시지
  "비즈니스 근육을 만드세요" vs "수익 파이프라인을 찾아드립니다"
  테스트: 두 버전 랜딩페이지 A/B
  GO 기준: 어느 쪽 이메일 수집이 1.5x 이상 높은지
  비용: $40 (광고)
```

## Wizard of Oz 가설 (만들기 전에 팔기)

> 핵심 원칙: 파이프라인을 만들지 않고, 최소 빌드로 "마법처럼 느끼게" 만들어 수요를 검증한다.
> 뒷단은 Claude API 1개 호출 + 수작업. 유저는 즉각적 도파민을 얻는다.

### UX 전략: "즉석 티저" 모델

```
기존 문제: "폼 채우고 48시간 기다려" → 도파민 없음. 이탈.
해결: 즉시 맛보기를 주고, 풀버전에 돈을 내게 한다.

핵심 UX 플로우:
  1. 유저가 관심 분야 입력 (입력창 1개, 한 줄)
  2. "AI가 분석 중..." 애니메이션 (20~30초, 실제 Claude API 호출)
  3. 즉시 티저 결과:
     ✅ 보이는 것: 니치 3개 이름 + 한 줄 요약 + 수요 신호 강도
     🔒 블러 처리: 시장 규모, 경쟁 분석, 검증 방법, 예상 수익
  4. CTA: "풀 리포트 받기" → 이메일 수집 or 결제

왜 강력한가:
  - "어 진짜 나오네?" 순간 = 즉각적 가치 체험
  - 블러 = "더 있는데..." 느낌 → 전환 유도
  - 공유 욕구: "나 이거 해봤는데 신기해" → 바이럴
  - 빌드량 최소: Next.js 1페이지 + Claude API 1개 + 블러 CSS
```

### UX 확장 옵션 (선택)

```
옵션 A: 니치 스코어보드 (1번 + α)
  - 이미 분석된 니치 10~20개를 카드 피드로 먼저 보여줌
    "AI 시니어 사진복원 — 🔥 수요 강함 — 경쟁 약함 — $2K MRR 가능"
    "비건 베이커리 사전주문 — ⚠️ 수요 중간 — 경쟁 강함"
  - 구경하다가 → "내 분야도 분석해줘" 클릭 → 즉석 티저로 연결
  - 장점: 진입 장벽 제로 (폼부터 안 시작), FOMO 유발, 캡쳐 공유 = 마케팅
  - 추가 빌드: 정적 니치 카드 10~20개 (Claude로 미리 생성)

옵션 B: 3문항 퀴즈 퍼널 (1번 + α)
  - "당신에게 맞는 사이드 비즈니스를 찾아드립니다"
    Q1: 관심 분야? (5개 선택지)
    Q2: 주당 투자 가능 시간? (3개 선택지)
    Q3: 초기 예산? (3개 선택지)
  - 즉시 결과: "당신은 [X 유형] — 이런 니치가 맞습니다"
  - 장점: 퀴즈 완료율 높음, "맞춤" 느낌, 결과 공유 욕구
  - 추가 빌드: 퀴즈 3스텝 UI

추천: 즉석 티저 단독으로 시작 → 데이터 보고 옵션 A or B 추가
```
