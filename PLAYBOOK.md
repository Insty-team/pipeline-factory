# Pipeline Factory 실행 플레이북

> Polsia 벤치마크 기반, "검증된 수익 파이프라인 발굴 서비스"
> Phase 0 (WoZ 검증): 2026-03-28 ~ 2026-04-27 (4주)
> Phase 1 (MVP 자동화): WoZ GO 판정 후 착수

---

## 서비스 철학

```
"영속적 사업 하나 찾는 것보다,
 필요할 때마다 작은 사업을 만들 수 있는 능력이 더 안전하다."

이 서비스는 도구가 아니라 "비즈니스 근육"을 만드는 과정이다.
```

### Polsia vs 우리의 본질적 차이

```
Polsia:  "AI가 당신 대신 회사를 운영합니다"
         → 유저는 수동적. 플랫폼 없으면 무력.
         → 복권 사는 구조. 대부분 $50 MRR 이하.

우리:    "작은 니치를 찾고 검증하는 법을 AI와 함께 반복하세요"
         → 유저는 능동적. 쓸수록 본인이 더 잘하게 됨.
         → 근육 만드는 구조. 반복할수록 성공률 상승.
```

### 자연스러운 종속 (강제 아닌 선택)

```
유저가 떠나지 않는 이유는 "못 떠나서"가 아니라:

1. 내 실험 히스토리가 여기 있음
   → 100번의 성공/실패 데이터 = 나만의 비즈니스 직관 DB
   → 다른 곳에서 처음부터 쌓기 싫음

2. 플랫폼 전체의 데이터를 누림
   → "이 니치는 지금 포화" 같은 정보는 혼자 못 얻음
   → 유저가 많을수록 내가 얻는 인사이트도 커짐

3. 내 근육이 이 도구에 최적화됨
   → 헬스장 바꾸면 기구 위치부터 다시 익혀야 하듯
   → "발견→검증→실행" 워크플로우가 몸에 밴 상태

4. 성공 경험이 쌓여있음
   → "여기서 3번째 파이프라인 성공했는데 왜 떠나?"
   → 성공한 기억이 있는 곳을 떠나기 어려움
```

## 서비스 컨셉

```
"AI가 실행만 하는 게 아니라, 뭘 해야 돈이 되는지 찾아준다"
"니치 타이밍 머신 — 언제 들어가고 언제 나올지를 데이터로"

Input:  관심 분야 / 보유 자원
Output: 검증된 수익 파이프라인 (데이터 포함)
```

### 셀링 메시지

```
메인:
  "대기업은 다음 먹거리 하나 찾는 데 수년, 수백억 씀.
   당신은 $20와 48시간이면 됨."

서브:
  "작은 니치 하나 찾아 $2K MRR 만드는 연습은
   MBA 2년보다 실전적이고, 창업 실패 1번보다 저렴하다."

금지 용어: AI, 자동화, 시스템, 파이프라인, 에이전트, 플랫폼
통하는 표현: "알아서 찾아줌", "48시간 안에", "데이터로 검증된"
→ WoZ Twitter 스레드, 랜딩페이지 카피에 직접 적용
```

### 니치는 고갈되지 않는다

```
니치 생성 속도 > 니치 포화 속도

이유:
1. 기술/문화/규제 변화 → 새 니치 폭발적 생성
2. 프랙탈 구조 → 상위 포화 시 하위 니치 개방
3. 같은 니치도 시장별(한/중/미) 다른 문제

핵심 가치: "포화된 니치를 빨리 버리고, 새 니치를 빨리 잡는 것"
= 속도 게임 = 데이터 많은 플랫폼이 이김
```

## 니치 발굴 듀얼 모드

> Pipeline Factory가 니치를 찾는 두 가지 방식. 배타적이 아니라 병렬 운영.

### 모드 A: 커뮤니티 기반 (새 시장 발굴)

```
Reddit, HN, Twitter 등에서 "아직 서비스가 없는" pain point 발굴
  → 시장 존재 여부가 불확실 → 랜딩페이지+광고로 수요 검증 필요
  → 신뢰도 시작점: 0.3~0.5
  → 리스크: 높음 (시장 자체가 없을 수 있음)
  → 리턴: 높음 (블루오션 선점 가능)
```

### 모드 B: 레퍼런스 기반 (기존 시장 차별화)

```
이미 돈 버는 서비스($10K+ MRR)를 레퍼런스로 삼고,
그 서비스의 불만/약점에서 차별화 가설을 세운다.
  → 시장 존재는 레퍼런스가 이미 증명 → 차별화만 검증하면 됨
  → 신뢰도 시작점: 0.6~0.8
  → 리스크: 낮음 (시장은 확실함)
  → 리턴: 중간 (경쟁자가 이미 존재)
```

#### 레퍼런스 Pain Point 수집 소스

```
1성 리뷰:    App Store, Google Play, G2, Capterra, Trustpilot
Reddit:     "X sucks because...", "looking for X alternative"
Twitter/X:  "@서비스명 complaint", "switched from X to..."
검색 데이터: "서비스명 alternative", "서비스명 vs", "서비스명 pricing too expensive"
취소 사유:   해지 페이지 패턴, "I cancelled X because..." 게시글
경쟁 비교:   "X vs Y" 블로그/유튜브에서 반복되는 약점
```

#### 레퍼런스 기반 가설 루프

```
Step 1: 레퍼런스 서비스 선정
  - 조건: $10K+ MRR, 리뷰 100개+, 1~3성 리뷰 비중 15%+
  - 소스: Product Hunt 상위, IndieHackers 수익 공개, SaaS 디렉토리

Step 2: 불만 크롤링 (AI 톤 분석)
  - 1~3성 리뷰 수집 → AI가 반복 패턴 추출
  - 카테고리화: 가격/UX/기능부족/고객지원/성능
  - 빈도순 정렬 → 상위 3개 pain point 선정

Step 3: 차별화 가설 3개 생성
  예시 (레퍼런스: Calendly):
    가설 H-010: "Calendly 무료 플랜 제한에 불만인 프리랜서 대상,
                 무제한 무료 + 프리미엄 $5/월 모델"
    가설 H-011: "Calendly 한국어 미지원 불만 →
                 카카오톡 연동 예약 서비스"
    가설 H-012: "Calendly UI 복잡하다는 불만 →
                 링크 1개로 끝나는 초간단 예약"

Step 4: 검증
  - Phase 0(WoZ): 수동 리포트로 검증 (티저 페이지는 모드 A 전용)
  - Phase 1+: 랜딩페이지+광고 자동화로 검증
  - 모드 B 장점: 레퍼런스 유저를 직접 타겟 광고 가능
    ("X가 불편하셨다면" 메시지, "X alternative" 키워드 SEO)
  - GO/NO-GO 판정 → 인사이트 추출 → 다음 가설
```

#### 모드 A vs 모드 B 비교

```
              모드 A (커뮤니티)     모드 B (레퍼런스)
시장 검증      필요                 불필요 (이미 증명됨)
경쟁          낮음 (블루오션)       있음 (레퍼런스가 경쟁자)
타겟팅        넓음 (관심 분야 기반)  정밀 ("X 불만 유저")
광고 메시지    "이런 니치가 있어요"   "X가 불편하셨다면"
SEO 기회      새 키워드 선점        "X alternative" 키워드
가설 신뢰도   0.3~0.5              0.6~0.8
추천 비중     30%                  70% (초기)

→ 초기에는 모드 B 70% + 모드 A 30%로 시작
→ 모드 B에서 감 잡으면 모드 A 비중 확대 (블루오션 탐색)
→ 두 모드의 인사이트는 같은 DB에 축적 → 교차 활용
```

## ⚡ Phase 0: Wizard of Oz 검증 (4주, 지금 시작)

> **이 섹션이 전체 플레이북의 출발점.** 아래 모든 장기 전략은 여기서 GO 판정이 나야 의미를 가진다.
>
> 핵심 원칙: 파이프라인을 만들지 않고, 최소 빌드로 "마법처럼 느끼게" 만들어 수요를 검증한다.
> 뒷단은 Claude API 1개 호출 + 수작업. 유저는 즉각적 도파민을 얻는다.

### UX 전략: "즉석 티저" 모델

```
기존 문제: "폼 채우고 48시간 기다려" → 도파민 없음. 이탈.
해결: 즉시 맛보기를 주고, 풀버전에 돈을 내게 한다.

핵심 UX 플로우:
  1. 유저가 관심 분야 입력 (입력창 1개, 한 줄)
  2. "AI가 분석 중..." 애니메이션 (20~30초, 실제 Claude API 호출)
  3. 즉시 티저 결과:
     ✅ 보이는 것: 니치 3개 이름 + 한 줄 요약 + 수요 신호 강도
     🔒 블러 처리: 시장 규모, 경쟁 분석, 검증 방법, 예상 수익
  4. CTA: "풀 리포트 받기" → 이메일 수집 or 결제

왜 강력한가:
  - "어 진짜 나오네?" 순간 = 즉각적 가치 체험
  - 블러 = "더 있는데..." 느낌 → 전환 유도
  - 공유 욕구: "나 이거 해봤는데 신기해" → 바이럴
  - 빌드량 최소: Next.js 1페이지 + Claude API 1개 + 블러 CSS
```

### 즉석 티저 — Claude API 호출 명세

```
Input: 유저가 입력한 관심 분야 (한 줄, 예: "pet grooming", "AI 교육")

Prompt 구조:
  "사용자의 관심 분야: {input}
   이 분야에서 지금 시작할 수 있는 니치 비즈니스 3개를 찾아줘.
   각 니치에 대해:
   1. 니치 이름 (한 줄)
   2. 왜 지금인지 (수요 신호, 한 줄)
   3. 수요 강도 (🔥강함 / ⚠️중간 / ❄️약함)
   4. [블러 처리될 내용] 시장 규모, 경쟁 분석, 검증 방법, 예상 MRR"

Output (유저에게 보이는 것):
  니치 1: "시니어 대상 AI 사진복원" — Reddit 관련 커뮤니티 6개월 40%↑ — 🔥
  니치 2: "반려동물 맞춤 영양 가이드" — 검색량 월 12K, 자동화 경쟁 없음 — ⚠️
  니치 3: "소규모 그루밍샵 예약 자동화" — 네이버 리뷰 불만 다수 — 🔥

Output (블러 처리):
  각 니치의 시장 규모, 경쟁사 3개, 검증 방법, 예상 MRR → 🔒

API 설정:
  모델: claude-sonnet-4-6 (속도 우선, 비용 절약)
  max_tokens: 1000
  예상 응답 시간: 10~20초
  예상 비용: ~$0.01/호출
```

### UX 확장 옵션 (데이터 보고 추가)

```
옵션 A: 니치 스코어보드
  - 이미 분석된 니치 10~20개를 카드 피드로 먼저 보여줌
    "AI 시니어 사진복원 — 🔥 수요 강함 — 경쟁 약함 — $2K MRR 가능"
    "비건 베이커리 사전주문 — ⚠️ 수요 중간 — 경쟁 강함"
  - 구경하다가 → "내 분야도 분석해줘" 클릭 → 즉석 티저로 연결
  - 장점: 진입 장벽 제로 (폼부터 안 시작), FOMO 유발, 캡쳐 공유 = 마케팅
  - 추가 빌드: 정적 니치 카드 10~20개 (Claude로 미리 생성)

옵션 B: 3문항 퀴즈 퍼널
  - "당신에게 맞는 사이드 비즈니스를 찾아드립니다"
    Q1: 관심 분야? (5개 선택지)
    Q2: 주당 투자 가능 시간? (3개 선택지)
    Q3: 초기 예산? (3개 선택지)
  - 즉시 결과: "당신은 [X 유형] — 이런 니치가 맞습니다"
  - 장점: 퀴즈 완료율 높음, "맞춤" 느낌, 결과 공유 욕구
  - 추가 빌드: 퀴즈 3스텝 UI

추천: 즉석 티저 단독으로 시작 → 데이터 보고 옵션 A or B 추가
```

### 운영 구조

```
[즉석 티저 — 자동]
  유저 입력 → Claude API → 티저 즉시 표시 (20~30초)

[풀 리포트 — 수동 (Wizard of Oz)]
  이메일/결제 전환된 유저만 대상
  │
  ├─ 내가 Claude로 심층 분석 (30분~1시간)
  │   ├─ Pain point 크롤링 결과 정리
  │   ├─ 가설 3개 상세 (경쟁, TAM, 검증 방법)
  │   └─ GO/NO-GO 기준 포함 리포트 작성
  │
  └─ 유저에게 전달 (이메일/Notion, 24시간 내)
     └─ 피드백 수집 → 다음 가설에 반영
```

### WoZ 가설 (순차 3단 검증)

> 즉석 티저 페이지가 트래픽 유입 + 무료 가치 체험의 입구.
> WoZ-001은 티저에서 "풀 리포트 받기" 클릭한 유저 대상으로 시작.

```
가설 WoZ-001: "무료 샘플 → 재신청 의향"
  오퍼: "관심 분야를 알려주시면, 48시간 내 검증된 니치 3개를 찾아드립니다 (무료)"
  채널: Twitter 스레드 + Indie Hackers 게시글
  수동 작업: Claude로 분석 → Notion 페이지로 전달
  측정:
    - 신청 수 (목표: 20건+)
    - 리포트 열람률 (목표: 70%+)
    - "다음에도 받고 싶다" 응답률 (목표: 50%+)
    - 자발적 공유/추천 발생 여부
  GO 기준: 신청 20건+ AND 재신청 의향 50%+
  비용: $0 (시간만)
  기간: 72시간

  → GO면 WoZ-002로 진행
  → NO-GO면: 오퍼 문구 or 채널 변경 후 재테스트

가설 WoZ-002: "유료 전환 가격 테스트"
  전제: WoZ-001에서 무료 수요 확인됨
  오퍼 A/B/C:
    A: $19 일회성 — "니치 리포트 1회"
    B: $29/월 — "주 1회 니치 리포트 구독"
    C: $49 일회성 — "니치 리포트 + 랜딩페이지 초안까지"
  채널: WoZ-001 무료 유저에게 DM + 새 트래픽 (Twitter/광고)
  수동 작업: 동일 (Claude 분석 + 수동 전달)
  측정:
    - 각 가격별 전환율
    - 결제 후 만족도 (NPS)
    - "이 가격이면 계속 쓸 의향" 비율
  GO 기준: 어느 가격이든 5건+ 결제
  비용: $20~30 (광고) + Stripe 수수료
  기간: 1주

  → GO면 WoZ-003으로 진행
  → NO-GO면: 가격 재조정 or 오퍼 구성 변경

가설 WoZ-003: "반복 구매 & 입소문"
  전제: WoZ-002에서 유료 전환 확인됨
  오퍼: WoZ-002에서 이긴 가격으로 고정
  추가 실험:
    - 리포트 받은 유저가 실제로 실행하는 비율
    - 실행 후 피드백 → 다음 리포트에 반영 (인사이트 루프 시작)
    - 레퍼럴: "친구 1명 초대하면 다음 리포트 무료"
  측정:
    - 재구매율 (목표: 40%+)
    - 레퍼럴 전환율 (목표: 초대 중 20%+ 신청)
    - 유저당 LTV 추정
  GO 기준: 재구매 40%+ OR 레퍼럴 20%+
  비용: ~$50 (광고 + 레퍼럴 무료분)
  기간: 2주

  → GO면: MVP 자동화 착수 (Phase 1)
  → NO-GO면: 오퍼/타겟/채널 중 하나 변경하여 다음 3개 가설 생성
```

### 확장 로드맵

#### 온라인 → 오프라인

```
Step A: 온라인 Only (Month 1~3)
  Reddit, Twitter, 커뮤니티 데이터 기반
  → 디지털 제품, SaaS, 온라인 서비스 파이프라인

Step B: Online-to-Offline (Month 3~6)
  온라인 수요 데이터로 오프라인 기회 추론
  예: "배달 음식 불만" → 특정 음식 카테고리 수요 발견
  AI: 리뷰 분석, 상권 데이터 API, 배달앱 수요 분석
  사람: 현장 답사, 파트너십, 물리적 프로토타입

Step C: Full O2O (Month 6+)
  1. AI가 오프라인 기회 발굴 (리뷰/데이터 분석)
  2. 온라인에서 수요 검증 (사전주문, 대기자명단)
  3. 검증되면 오프라인 실행 가이드 제공
  4. 오프라인 운영 중 온라인 마케팅 자동화
```

```
현재 상태: 보류 (시장 미성숙)

- Physical AI (navigation & manipulation) 도메인 현직 경험 보유
```

## 매출 경로 & 퇴사 판단

> 환율 기준: $1 = ₩1,350 (2026년 3월 기준). Month 1 = Phase 0 시작 시점(2026-03-28).

### 서비스 매출 (Pipeline Factory 자체)

```
경로 1: 구독 모델
  $29/월 × 700명 = $20,300/월 ≈ 2,700만원
  $49/월 × 450명 = $22,050/월 ≈ 3,000만원
  현실성: 단독으로 2~3천만원 = Month 7~13 시나리오

경로 2: 하이터치 + 구독 혼합
  베이직 $29/월 × 200명      = $5,800
  프로 $99/월 × 50명          = $4,950
  에이전시 $299/월 × 10팀     = $2,990
  일회성 리포트 $49 × 100건/월 = $4,900
  합계: $18,640/월 ≈ 2,500만원
  현실성: Month 5~7 시나리오

경로 3: 마켓플레이스 수수료 (장기)
  검증된 파이프라인 실행 연결 → 성공 수수료 10~20%
  월 $10K 파이프라인 10건 × 15% = $15,000 + 구독 $5,000
  합계: $20,000/월 ≈ 2,700만원
```

### "능력" 자체의 매출

```
Pipeline Factory 운영으로 쌓이는 "니치 발굴/검증 근육":

1. 직접 실행: 검증 통과 니치 직접 운영 → $3K~10K MRR × 2~3개
2. 컨설팅: 니치 검증 전문가 포지셔닝 → $2K~5K/건 × 3~5건/월
3. 콘텐츠/교육: 실험 과정이 콘텐츠 → 코스 $199~499/인

종합: 서비스 $10K + 직접 실행 $10K + 컨설팅 $5K = $25K/월 ≈ 3,300만원
→ "능력"이 여러 수익원을 만드는 구조 = "비즈니스 근육" 철학과 일치
```

### 이전 전략에서 가져온 핵심 인사이트

```
1. "GPT로 해결 불가능한 것"이 진짜 기회
   - GPT = 텍스트 생성. 시스템 통합, 실행, 인간관계는 못 함.
   - Pipeline Factory가 찾아줄 파이프라인 = 이 갭을 메우는 것들.

2. 1 Action = 3 Layers of Value
   - 돈 (단기 현금) + 콘텐츠 소재 (중기 자산) + 인사이트 DB (장기 해자)
   → WoZ 리포트 1건 = 매출 + 트위터 소재 + 인사이트 DB

3. 기술 용어 배제 → 셀링 메시지 섹션에 반영 완료

4. 저가 다수 > 고가 소수 (초기)
   - 저가 = 진입 허들 낮음, 입소문 빠름, 이탈률 낮음
   → WoZ에서 $19~29 먼저 검증하는 이유

5. 데이터가 해자
   - 가설 1000개 축적 = 복제 불가능
   - Reddit pain point 연구(9,363 포스트) = 첫 인사이트 DB 시드

6. 레퍼런스 기반(모드 B)이 초기 성공률 높음
   - 시장 존재가 이미 증명됨 → 차별화만 검증
   - 초기 70% 모드 B + 30% 모드 A로 시작 → 감 잡으면 비중 조정
```
