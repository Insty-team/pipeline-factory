# GEO Truth Layer — 포지셔닝 + 영업 메시지 v2

> 2026-04-20 AEO/GEO 업데이트: 고객-facing 정의를 **AI가 병원을 정확히 찾고 설명하게 만드는 공식 정보 운영 서비스**로 둔다. AEO/GEO는 시장/내부 용어이며, 병원에게는 `AI가 병원 공식 정보를 헷갈리지 않게 만드는 작업`으로 설명한다.

Date: 2026-04-04
Core Belief: 잘못된 노출은 클리닉이든 환자든 피곤하게 만들 뿐이다.

---

## 한 줄 포지셔닝

> **AI가 병원을 정확히 찾고 설명하게 만드는 공식 정보 운영 서비스**

> "AI 검색에 더 많이 노출시키는 게 아니라, 정확하게 노출시킵니다.
> 맞는 환자가 맞는 클리닉을 찾도록."

---

## 기존 GEO vs Truth Layer GEO

| | 기존 GEO 업체 | 우리 (Truth Layer) |
|---|-------------|-------------------|
| 목표 | AI에 많이 노출 | AI에 **정확하게** 노출 |
| 약속 | "나오게 해드립니다" | "맞는 환자가 찾아오게 합니다" |
| 콘텐츠 | 마케팅 카피 최적화 | 공식 소스 기반 팩트만 |
| 결과 | 노출 수 ↑ | 매칭 정확도 ↑, 상담 피로 ↓ |
| 리스크 | 과장 → 기대 불일치 → 불만 | 정확 → 맞는 기대 → 만족 |

## 왜 "정확한 노출"인가

### 잘못된 노출이 만드는 문제

**환자 쪽:**
- AI가 가격 29,000원이라 했는데 가보니 다름
- "내 피부에 맞는지" 정보 없이 가서 상담에서 돌아옴
- 여러 클리닉 비교하고 싶은데 정보 기준이 다 다름
- → 피곤, 불신, 이탈

**클리닉 쪽:**
- 잘못된 기대 가진 환자가 옴 → 상담 시간 낭비
- "온라인에서 봤는데 다르다" 컴플레인
- 가격/시술 정보 왜곡 → 경쟁사와 잘못된 비교
- → 피곤, CS 부담, 리뷰 악화

### 정확한 노출이 해결하는 것

**환자 쪽:**
- AI가 알려주는 정보 = 공식 소스 기반 = 신뢰
- "이 시술은 이런 분께 맞습니다" → 사전 셀프 필터링
- 가격/조건/제한사항까지 구조화 → 비교 가능
- → 맞는 클리닉에 맞는 기대로 방문

**클리닉 쪽:**
- 맞는 환자가 옴 → 상담 효율 ↑
- 정보가 정확하니까 "다르다" 컴플레인 ↓
- AI가 공식 정보 인용 → 브랜드 신뢰 ↑
- → 시술 만족도 ↑, 리뷰 개선, 재방문 ↑

## 세일즈 메시지 v2 (정확한 매칭 강조)

### 핵심 피치 (30초)

> "사장님, AI 검색에 노출만 시키면 맞지 않는 환자도 옵니다.
> 저희는 사장님 클리닉의 공식 정보를 AI가 정확하게 읽을 수 있게 구조화합니다.
> 그러면 맞는 환자가 맞는 기대를 갖고 옵니다.
> 상담 시간 줄고, 만족도 올라가고, 리뷰가 좋아집니다."

### 반론 대응 v2

**"노출만 늘리면 되는 거 아닌가요?"**
> "노출만 늘리면 안 맞는 환자도 같이 늘어납니다. '온라인에서 봤는데 다르다'는 컴플레인 받아보셨죠? 정확한 정보로 노출해야 맞는 환자가 옵니다."

**"우리 정보가 뭐가 부정확한데?"**
> "사장님 홈페이지 정보는 정확합니다. 문제는 AI가 그걸 못 읽는 거예요. 여러 페이지에 흩어진 정보를 AI가 엉뚱하게 조합하거나, 아예 무시합니다. 저희가 하는 건 사장님 공식 정보를 AI가 정확하게 읽을 수 있는 형태로 정리하는 겁니다."

**"그래서 환자가 더 오긴 해요?"**
> "노출 보장은 못 합니다. 대신 확실한 건, 잘못된 정보로 온 환자보다 정확한 정보로 온 환자가 시술까지 갈 확률이 훨씬 높다는 겁니다. 광고는 많이 오게 하고, 저희는 맞게 오게 합니다."

무료로 현황 분석 리포트 보내드릴 수 있는데
관심 있으시면 편하게 답변 주세요!

## 서비스명 후보

"GEO 대행" 이라고 하면 경쟁사와 구분이 안 됨.
서비스명에 차별점이 드러나야 함.

| 후보 | 느낌 |
|------|------|
| **정확한 노출** | 직관적, 한국어 자연스러움 |
| **AI 매칭 최적화** | 기술적, B2B 느낌 |
| **Truth Layer** | 영어, 글로벌 확장 시 |
| **매치핏 (MatchFit)** | 매칭+적합, 브랜드화 가능 |
| **팩트레이어 (FactLayer)** | 팩트 기반 강조 |

## 이 포지셔닝이 매각 가치도 높이는 이유

```
기존 GEO 업체:    "노출 최적화 기술" → 기술은 대체 가능
우리 (Truth Layer): "검증된 의료 정보 데이터셋" → 데이터는 대체 불가

인수자가 원하는 것:
├── 강남언니/모두닥: 정확한 클리닉 정보 DB
├── 대형 대행사: 의료 버티컬 전문성 + 고객 관계
├── 해외 GEO사: 한국 의료관광 시장 데이터
└── 전부 "데이터"가 핵심
```

클리닉 1곳의 구조화 데이터를 만들 때마다
→ 매출 + 데이터 자산이 동시에 쌓이는 구조
