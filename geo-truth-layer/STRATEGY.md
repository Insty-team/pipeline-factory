## 0. 한 줄 요약

**짧게**: 클리닉 GEO 컨설팅(현재 가설)으로 6개월 안에 월 200~300만원 만들고, 그동안 축적되는 데이터·관계·표준을 wedge 삼아 36개월 안에 **한국 의료 AI 검색 인프라(데이터·표준 layer)** 포지션을 노린다.

**길게**: 서비스업 자체는 1 to N이라 Zero to One이 아니다. 그러나 50~100곳 클리닉을 감사하면서 축적되는 (a) 한국 의료 웹스택 패턴, (b) AI 모델별 한국 의료 쿼리 실패 모드, (c) 의료법 호환 구조화 데이터 패턴, (d) 클리닉 사장 의사결정 모델 — 4개 secret 가설이 실증되면, 그것을 SaaS화 → 표준화 → 데이터 utility로 확장하는 경로가 존재한다. 핵심 변곡점은 (1) 가설 검증, (2) productize 결단, (3) cofounder 합류.

## 1. 단기 게임 — 월 200~300만원 (지금~6개월)

### 1.1 수익 산수

| 모델 | 200만원 조합 | 300만원 조합 |
|---|---|---|
| Setup만 (40만원) | 5건/월 | 7~8건/월 |
| Monthly만 (17만원 누적) | 12건 active | 18건 active |
| **현실 믹스** | Setup 2 + Monthly 8 | Setup 3 + Monthly 12 |

→ **Setup 신규 + Monthly 누적**의 복리 구조가 유일한 현실 경로. Setup 단독은 매달 5건 클로즈 압박으로 지속 불가능.

### 1.2 Funnel 역산 (Setup 3건/월 기준)

```
Setup 3건/월
  ↑ 클로즈율 50%
파일럿 6건/월
  ↑ 전환율 30%
응답 20건/월
  ↑ 응답률 15%
발송 130곳/월 ≈ 주당 30~35곳
```

병목 우선순위: **발송량 > 응답률 > 전환율 > Monthly 유지**.

### 1.3 단기 4단계 (Phase 0~3)

| Phase | 기간 | 매출 목표 | 핵심 산출물 | Go/No-Go |
|---|---|---|---|---|
| **P0 가설 검증** | 지금~5/10 (2주) | 0 | Wave 1 응답률 데이터, 1차 무료 파일럿 시작 | 응답률 ≥15% → P1 / <5% → 채널·메시지 피벗 |
| **P1 케이스 스터디 1건** | 5/10~6/7 (4주) | 30~50만 | 무료 파일럿 2~3건, before/after AI 검색 결과 캡처, 첫 유료 Setup | 케이스 1건 미확보 시 P2 진입 보류 |
| **P2 발송 스케일업** | 6월 (4~6주) | 100~150만 | 시각자료 v3.x 템플릿화, 주당 30곳 발송 체제 | Setup 2~3건/월 + Monthly 1~2건 안 나오면 가격·세그먼트 재설계 |
| **P3 Monthly 누적** | 7~9월 | **200~300만** | Setup 2~3건 + Monthly 10~15건 active, 자동화(주간 검색 모니터링·월 리포트) 시작 | 누적 12건 미만이면 churn 원인 분석 |

### 1.4 단기 병목과 대응

| 병목 | 대응 |
|---|---|
| 응답률 (현재 0/4) | Wave 1 v3.1 시각자료 가설 검증 → 실패 시 채널 전환(전화/인스타 DM) |
| 발송량 → 시간 부족 | v3.1 HTML을 변수 기반 템플릿화 (1곳당 60분 → 15분), 강남/명동 100곳 사전 DB |
| 유료 전환 (파일럿→돈) | 파일럿 시작 시점에 약정 명시: "4주 후 결과 좋으면 setup 40만 + monthly 15만" |
| Monthly churn | Setup 결과물 안에 "주간 AI 검색 모니터링 자동 리포트"를 prebuilt 포함, 가치 체험시키기 |

### 1.5 가격·패키지 재설계 (케이스 스터디 1건 후)

| 기존 (검증 단계) | 케이스 후 (P2~P3) |
|---|---|
| Setup 30~50만 | **Setup 80만 (첫 3개월 monthly 무료 포함 약정)** |
| Monthly 15~20만 | Monthly 25만 (4개월차부터) |

같은 12건 active = 300만. **가격 인상 못 하면 영원히 발송량 게임**.

## 2. 장기 게임 — Zero to One (6개월~36개월)

### 2.1 솔직한 출발점

현재 형태(클리닉 GEO 컨설팅) **자체는 Zero to One이 아니다**.
- 진입장벽 거의 없음 (Korean SEO 에이전시가 12~24개월 안에 같은 서비스 추가)
- 누적 자산 = 시간×인력 (스케일 = 인력 비례)
- 솔로 천장: 연 ~5억, 팀 ~50억급 SEO 에이전시
- 남이 못 들어오는 구조적 이유 부재

→ **이 형태로 200~300만원은 가능, Zero to One은 불가능**. 이 서비스를 **wedge**로 들어가서 별도 게임을 차지해야 함.

### 2.2 Thiel 7질문 점검

| 질문 | 현재 | Zero to One 가능 답 |
|---|---|---|
| Engineering: 10x 더 나은 기술? | 수동 + 범용 SEO | 한국 의료 특화 검증·스키마 표준 + 자동화 |
| Timing: 지금이 적기? | ✅ 18~36개월 창 | 동일 |
| Monopoly: 작은 시장의 큰 점유? | 가능 (강남/명동 클리닉 GEO) | 한국 의료 AI 검색 인프라 |
| People: 팀? | ❌ 솔로 | P2에 cofounder 필수 |
| Distribution: 전달 경로? | 카톡 콜드 DM | 의료협회/마케팅 에이전시/장기 인바운드 |
| Durability: 10년 뒤에도? | ❌ 서비스는 복제 쉬움 | 데이터+표준+관계 누적은 가능 |
| **Secret**: 남이 못 보는 것? | ⚠️ 아직 가설 | ↓ 다음 섹션 |

### 2.3 Secret 가설 4개 (검증 대상)

50곳 감사하면서 *남들보다 먼저* 축적될 수 있는 것:

1. **한국 의료 웹스택 특화 패턴** — imweb·메이크샵·자체 빌더의 AI 검색 실패 모드. Korean SEO 에이전시도 아직 카탈로그화 안 함.
2. **AI 모델별 한국 의료 쿼리 동작** — ChatGPT/Perplexity/네이버Cue가 어떻게 헷갈리는지의 ground truth.
3. **의료법·광고법 호환 구조화 데이터 패턴** — 다른 버티컬 SEO는 이 제약 모름.
4. **클리닉 사장 의사결정 모델** — 어떤 메시지에 돈을 쓰는지의 실전 데이터.

이 4개의 **교집합**이 secret. 외국 회사는 한국 의료 규제 때문에 못 들어오고, 한국 SEO 에이전시는 의료법을 모르고, 의료 마케팅 에이전시는 AI 검색을 모름.

**검증 방법**: 50곳 감사 후 패턴이 수렴하는지로 판정. 안 수렴하면 Zero to One 길은 닫히고 일반 SEO 에이전시 길만 남음.

### 2.4 Zero to One 시나리오 4개

| 경로 | 본질 | 해자 | 천장 | 솔로 가능성 |
|---|---|---|---|---|
| **A. 한국 의료 GEO SaaS** | Vertical SaaS | 통합 + 의료법 모듈 | 수백억 | P2까지 OK |
| **B. 한국 임상 데이터 API** | Data utility (네이버Cue/Perplexity가 cite하는 ground truth) | 데이터 깊이 + 검증 신뢰 | 수천억 | 어려움 (자본·팀 필요) |
| **C. 의료 GEO 표준 + 인증기관** | Standard setter | 제도 채택 | 한정적 | OK |
| **D. AI Clinic Ops 플랫폼** | 클리닉용 AI 운영 (검색+챗봇+예약+준수) | end-to-end 통합 | 수천억 | 어려움 |

**가장 현실적인 합성**: **A → B 경로** (SaaS로 클리닉 lock-in → 데이터 자산 누적 → API/utility로 확장).
- A 단독: 평범한 vertical SaaS (1 to N에 가깝다)
- A + B 결합: 클리닉 데이터를 *유일하게 보유* → 강력한 해자

### 2.5 36개월 5단계 로드맵

```
Phase 0 (지금~6개월) — 서비스 wedge로 검증
  목표: 클리닉 30~50곳, 월 200~300만원
  진짜 산출물: 패턴 데이터셋 + 사장 의사결정 모델 + 케이스 스터디 3건
  검증: secret 4개 가설이 실증되는가?

Phase 1 (6~12개월) — SaaS Productize
  목표: GEO 감사·모니터링·리포트 자동화 = 클리닉용 self-serve SaaS
  매출: 월 1,000~3,000만원 (관리 50~150곳)
  필요: cofounder (sales) 또는 첫 직원
  진짜 산출물: "한국 의료 GEO 표준 v1" 공개

Phase 2 (12~24개월) — 표준화 + 협회 채택
  목표: 의료협회/마케팅 에이전시 파트너십, 표준 공식화
  매출: 월 5,000만원~1억 (관리 300곳+ + 표준 채택)
  진짜 산출물: 데이터셋 1,000곳+, 협회 endorsement 1~2건

Phase 3 (24~36개월) — 데이터 layer 오픈
  목표: API로 AI 엔진·비교사이트·정부에 데이터 공급
  매출: 월 1~5억 (서비스+SaaS+데이터 라이선스 믹스)
  진짜 산출물: 한국 의료 GEO 인프라의 default 위치

Phase 4 (36개월+) — 인접 버티컬·지역
  치과·한의원·종합병원 → 같은 playbook
  동남아·일본 의료 시장 → Korean playbook export
```

### 2.6 Zero to One이 가능하기 위한 3대 조건

1. **6개월 안에 secret 4개 중 최소 2개 실증** — 단순 가설로 끝나면 그냥 SEO 에이전시
2. **12개월 안에 cofounder/첫 팀원 합류** — 솔로로는 P1 끝까지 못 감 (sales와 dev/ops 동시 필요)
3. **P0→P1에서 productize 결단을 미루지 않기** — 서비스 매출 안정화되면 인간은 productize 미루는 게 디폴트, 가장 흔한 함정

### 2.7 피벗·강등 트리거

| 시나리오 | 결과 |
|---|---|
| Wave 1~3에서 응답률·전환율 한계를 못 넘김 | Wedge 자체가 약함, 다른 진입점 탐색 |
| 50곳 감사해도 패턴이 안 수렴 | Secret 가설 실패, **일반 SEO 에이전시 길로 강등** |
| 강남언니가 GEO 기능 직접 추가 | 클리닉 측 layer 무력, **B(데이터 layer)로 강제 우회** |
| 네이버Cue/ChatGPT가 클리닉 검증 직접 빌드 | A 경로 위협, B/C(표준+API)가 유일 출구 |

## 3. Zero to One 노릴 때 *지금* 해야 하는 추가 행동

200만원 목표만이면 발송량+케이스 스터디에 집중해도 됨. Zero to One 노린다면 **추가로**:

| 행동 | 이유 | 시작 시점 |
|---|---|---|
| 모든 감사를 **구조화 데이터로 저장** (Markdown 말고 Postgres/JSON 스키마) | 데이터셋이 미래의 자산 — 지금 형식이 미래 API의 골격 | 즉시 |
| **첫 20곳을 거래 아닌 파트너로 대우** | 후기·소개·데이터 권한·장기 lock-in 모두 여기서 나옴 | P0 |
| **의료 GEO 표준 초안 공개 글 1편** (블로그/Medium) | 표준을 쓰는 자가 표준이 됨, 선점 효과 | P0 후반 |
| **6개월 시점 cofounder 탐색 시작** | 솔로 한계 직면 전에 미리 — 보통 1년 걸림 | 5/M+ |
| **두 번째 버티컬 가설 1개 탐색** (치과/한의원) | "의료 전체로 일반화되는가" 시그널 | P1 |
| **강남언니/네이버Cue/Kakao 동향 모니터링** (월 1회) | 위에서 들어오면 게임 끝나는 layer 추적 | 즉시, 월 1회 |

## 5. 핵심 메시지 — 이 문서가 사라져도 기억할 것

1. **현재 형태는 Zero to One이 아니다**. Wedge일 뿐.
2. **진짜 게임은 데이터·표준 layer**. 50곳 감사하면서 자산이 쌓이는지를 본다.
3. **단기 200~300만원과 장기 Zero to One은 같은 행동의 다른 시간대 보상**. 한쪽만 보면 둘 다 놓침.
4. **3개의 변곡점**: 가설 검증 → productize 결단 → cofounder 합류. 이 셋 중 하나만 어긋나도 일반 SEO 에이전시로 강등.
5. **분기마다 이 문서를 다시 펼친다**. 행동이 가설을 따라가고 있는지, 가설이 환경에 맞춰 갱신되고 있는지.
