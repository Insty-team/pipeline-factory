# 2026-04-20 AEO/GEO 인사이트 — 공식 정보 운영 서비스 가설

## 오늘 얻은 핵심 판단

GeekNews Weekly의 AEO(Agentic Engine Optimization) 흐름을 읽고, 기존 `GEO Truth Layer` 가설을 다음처럼 재정의한다.

> **AI가 병원을 정확히 찾고 설명하게 만드는 공식 정보 운영 서비스**

기존에 우리가 `GEO`라고 부르던 것은 시장에서는 `AEO`, `AAO`, `Agent-ready`, `AI visibility`와 섞여 쓰이고 있다. 완전히 같은 말은 아니지만, 실무적으로는 같은 문제군을 다룬다.

- **GEO**: ChatGPT, Perplexity, Gemini, Google AI Overviews 같은 생성형 답변에 병원이 정확히 인용/추천되는가
- **AEO**: AI agent가 병원 정보를 발견하고, 읽고, 이해하고, 다음 행동에 활용할 수 있는가
- **우리 포지션**: 클리닉 대상 `GEO + AEO`를 병원 언어로 번역한 **공식 정보 운영 레이어**

## Before / After 프레임

### 기존 병원 정보 운영

```
홈페이지/네이버플레이스/블로그/SNS에 정보가 흩어짐
→ AI가 못 읽거나, 지점/가격/시술/주소를 헷갈림
→ 원장/실무자는 왜 AI 검색에 안 뜨는지 모름
→ 가끔 수정하지만 결과 추적 없음
```

### GEO Truth Layer 이후

```
공식 URL 전달
→ AI/AEO 관점 감사
→ 병원 공식 정보 검수
→ AI-readable 구조화(schema, robots, sitemap, llms.txt, 정적 프로필/Markdown 등)
→ 공개 배포
→ 핵심 쿼리별 노출/설명 정확도 추적
```

## 고객에게 말할 문장

전문 용어를 앞세우지 않는다. 병원 대상 메시지는 아래 문장으로 통일한다.

> **ChatGPT 같은 AI가 병원을 잘못 이해하거나 아예 못 찾는 일이 없도록, 공식 정보를 정리하고 AI가 읽기 쉬운 형태로 배포·추적해드립니다.**

더 짧게는:

> **AI가 우리 병원을 헷갈리지 않게 만드는 작업입니다.**

## 이번 인사이트가 바꾸는 것

1. `AI 검색 노출 대행`이 아니라 **공식 정보 운영 서비스**로 설명한다.
2. `노출 수`보다 **설명 정확도, 맞는 환자 매칭, 헛문의 감소**를 전면에 둔다.
3. `llms.txt`, `schema`, `robots.txt`는 고객에게 팔 단어가 아니라 내부 실행 항목으로 둔다.
4. 병원에는 “AEO/GEO”보다 **AI 검색 현황 / 공식 정보 정확도 / AI가 읽는 구조**로 말한다.
5. 초기 검증은 여전히 1개 lane: **아웃리치 반응 테스트**다.

## 오늘 20~30분 액션 추천

**유스타일나인 1곳만 테스트한다.**

목표는 계약이 아니라 `무료로 짧게 보내주세요` 또는 `무슨 문제인가요?` 반응을 얻는 것이다.

1. ChatGPT/Gemini 중 하나에서 `강남 레이저 제모 추천` 또는 `강남 피부과 제모 추천` 검색 결과 스크린샷 1장 저장 — 5분
2. 유스타일나인 홈페이지/robots 상태 근거 1줄 메모 — 5분
3. 아래 메시지로 카카오톡/문자/전화 가능한 채널 1곳에만 발송 — 5분
4. `무응답 / 관심 / 거절 / 추가질문` 중 하나로 기록 — 5분

복붙 메시지:

```text
안녕하세요. AI 검색에서 병원 정보가 어떻게 보이는지 확인하다가 연락드립니다.

요즘 고객들이 ChatGPT나 Gemini로 “강남 레이저 제모 추천”처럼 검색하는데,
유스타일나인의원은 관련 검색에서 잘 잡히지 않는 편입니다.

확인해보니 홈페이지 설정상 AI가 병원 정보를 읽기 어려운 부분이 있어 보여서요.
정확히 말하면 “노출을 늘리는 작업”이라기보다, AI가 병원 공식 정보를 헷갈리지 않게 만드는 작업입니다.

원하시면 무료로 아주 짧게 현황만 정리해서 보내드릴게요.
```
